# 2026 世界杯 — football-odds-model 全量回测 + 改进建议（截至 6/21,40 场)

> 用模型 **v3.2 当前默认引擎**(Elo + 主场 +85 → λ → Poisson + Dixon‑Coles + draw_boost 0.06)逐场赛前预测全部 **40 场已赛**比赛,再与真实结果对比(比分概率、胜平负结果与概率、大小球)。脚本:`backtest_v33.py`、`verify_fixes.py`。
> 6/22 的阿根廷‑奥地利、法国‑伊拉克、挪威‑塞内加尔仍在进行,未纳入。
> 教育/分析用途,不构成投注建议。

## 一、总体回测指标(40 场)

| 指标 | 结果 | 解读 |
|---|---|---|
| 胜平负 — argmax(只判 1/2) | **23/40 = 57%** | 单场足球的合理上限(55–60%) |
| 胜平负 — 含平局规则(Tipset) | **17/40 = 42%** | ⚠️ 平局规则**反而拉低**了硬命中率 |
| RPS(越低越好) | **0.1639** | 概率校准良好 |
| 比分对数似然合计 | **−117.56** | draw_boost=0.06 处最优 |
| 真实平局率 | 13/40 = **32.5%** | 模型平均平局概率 **29.0%**(已贴近) |
| 大比分(净≥3) | 真实 **11** vs 模型期望 **7.1** | 仍系统性少算 |
| 大小球 2.5 Brier | **0.2545** | ⚠️ 比掷硬币(0.25)还差 |

**一句话诊断:概率校准(RPS/平局率)其实不错,但三处把概率转成"单一决策"的地方有系统偏差——平局硬选、大小球倾向、大比分右尾。**

## 二、做得好的部分(继续保留)

1. **胜负方向 57%** 已是单场上限;顶级热门基本兑现(阿根廷、法国、英格兰、德国两场、挪威、西班牙第二场 4‑0)。
2. **压比分精准命中**:墨西哥 1‑0 韩国、美国 2‑0 澳洲、摩洛哥 1‑0 苏格兰、巴西 3‑0 海地、西班牙 4‑0 沙特等,首选比分直接命中。
3. **draw_boost=0.06 经回测确为最优**:把模型平均平局概率从 23% 抬到 29%,贴近真实 32.5%,RPS 0.1649→0.1639。
4. **DC 低分修正**让 1‑1 / 0‑0 / 1‑0 这些主力小比分概率合理。

## 三、三个系统性缺陷(本次回测新确认)

### 缺陷 1 ⭐ 大比分右尾仍偏瘦(可改,已验证修法)
最差校准的 8 场全是大比分:德国 7‑1(P=0.1%)、瑞典 5‑1、加拿大 6‑0、荷兰 5‑1、英格兰 4‑2、日本 4‑0、瑞士 4‑1、美国 4‑1。纯 Poisson 假设进球独立,算不出"打开后扎堆"。

**验证的修法:ΔElo 大时自动触发 `--opp-style open`(无需手动标记)**

| 自动触发阈值 | 比分 logL | RPS |
|---|---|---|
| 默认(不触发) | −117.56 | 0.1639 |
| ΔElo ≥ 350 | **−116.47** | 0.1630 |
| ΔElo ≥ 300 | −116.54 | **0.1626** |

→ **建议:把 `--opp-style open` 在 (有效 Elo 差 ≥ ~300–350) 时自动开启**,logL 与 RPS 同时变好,且不必靠结果倒推标记。仍救不全(德国 7‑1 那种超神终结是不可约方差)。

### 缺陷 2 ⭐ 平局"硬选"规则过度触发(应改回报概率,不硬选)
draw_boost 修好了平局**概率**,但 Tipset 把它转成硬选 X 时,在宽门槛(gate<0.52)下**40 场里判平 20 次,只对 5 次**,把命中率从 57% 砸到 42%。

| 平局规则门槛 | 命中率 | 判平次数 / 命中 |
|---|---|---|
| gate < 0.52(现状) | 17/40 = 42% | 20 / 5 |
| gate < 0.45 | 20/40 = 50% | 7 / 1 |
| **gate < 0.42** | **23/40 = 57%** | 4 / 1 |

→ **建议:单一硬选回到 argmax(或把 gate 收紧到 ~0.42)。** 平局应当作**概率**呈现(RPS 已经很好),而不是逼模型在"铁桶逼平大热门"这种不可约平局上硬猜——任何规则都抓不到它。

### 缺陷 3 ⭐ 大小球 2.5 系统性偏"小"(可改,已验证修法)
模型对**全部 40 场**都给出 P(Over)<50%(全部判"小"),但真实 21/40=52.5% 走大,Brier 0.2545 比掷硬币还差。两个原因:base 进球水平(2.6)偏低 + draw_boost 把概率挪向低分对角线、顺带压低了 Over。

| 总进球基准 avg_goals | 平均 P(Over) | Brier |
|---|---|---|
| 2.6(现状) | 45.9% | 0.2545 |
| **2.8** | **50.7%** | **0.2500** |
| 3.0 | 55.3% | 0.2505 |

(且实测:draw_boost 0→0.06 会把 P(Over) 从 48.2% 压到 45.9%。)

→ **建议:把 avg_goals 基准从 2.6 上调到 ~2.8**(贴合本届 2.7 球/场的实际节奏),并让 draw 膨胀对"总进球中性"——只在平/非平之间挪概率,不顺手压低 Over。

## 四、改进清单(v3.3,按性价比排序)

1. **自动大比分尾巴**:`--opp-style open` 在有效 Elo 差 ≥ 300–350 时自动开启。**已回测验证**:logL −117.56→−116.47,RPS 0.1639→0.1626。
2. **平局只报概率、不硬选**:Tipset 单选回 argmax(或 gate 收到 0.42)。**已回测验证**:硬命中 42%→57%。
3. **总进球基准上调到 ~2.8**:**已回测验证**:O/U Brier 0.2545→0.2500,且让 draw 膨胀保持总进球中性。
4. **保留并继续接受的不可约方差**:大热门被低位铁桶逼平(西班牙 0‑0 佛得角、乌拉圭 2‑2 佛得角)、超神终结型大胜(德国 7‑1)——只用概率表达,不强行单点预测。
5. **数据层(已知改进方向)**:接入临场 xG 做贝叶斯更新、确认首发、收盘盘口去 margin 作锚——比任何参数微调都更能提升单场精度。

> 核心结论:模型的**概率校准本身是好的**(RPS 0.164、平局率已贴近)。本次回测最大的价值是定位了三处"概率→决策"的偏差,其中两处(自动加尾、O/U 基准)和一处(平局别硬选)都已用 40 场数据验证了正确改法。
